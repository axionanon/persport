Websites

There have been three iterations of my personal website. I am currently working building another one from scratch, a site that utilises Java and has an overall smoother interface.
I used to run this website you (my lovely visitor) are on right now on my university server. I decided to move everything to github recently! After this move I added a few of my most recent personal projects to my main page and the little ticker tape message you see at the top here :)

v2 of my site (this one) is rather basic. I've used some style sheets from W3 schools and did my own tinkering with the Personal Projects section and image gallery. All of the .html pages one might visit on this website are results of me experimenting with how to present material on a webpage. Whether that be merely inserting pdfs with iframe src or reference links (admittedly a bit of a lazy way to display content, but everyone has to start somewhere right?). There are still some other examples of visual media/graphic design I plan on adding soon..to my new site of course.

The biggest issue I'm dealing with on my new site is not the code itself—which I still do have my fair share of problems with—but using hugo with Github pages. For the newest site, I wanted my code to live in a private repo, where sensitive data and personal information could be protected. Using hugo, the plan is to push all the content from my private repo to a public one, where I can run Github pages and publish. I've cloned the repo using my local terminal successfully, but I'm currently working through issues with deployment and getting hugo to work effectively with my site. I think it might be because my computer has a tendency to save files in strange places that makes retrieving difficult, but I digress. I'm also trying to implement the Intersection Observer API.
